# GitHub Actions CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a GitHub Actions CI workflow for `eslint-lensflow-plugin` that enforces clean compilation, ESLint + oxlint linting, Prettier formatting, and coverage-gated tests on every PR and push to `main`.

**Architecture:** A single `ci` job in `.github/workflows/ci.yml`, scoped to `eslint-lensflow-plugin/` via `working-directory`, running `npm ci` then a fail-fast sequence of checks (typecheck → build → lint → oxlint → format check → test+coverage). Supporting tooling (Prettier, oxlint, vitest coverage) is added to the package first and proven to work locally before the workflow file wires them together. Coverage thresholds are pinned to the measured baseline.

**Tech Stack:** GitHub Actions (`actions/checkout@v7`, `actions/setup-node@v7`, `actions/upload-artifact@v7`), Node 24.x, npm, vitest + `@vitest/coverage-v8`, Prettier, oxlint, Dependabot.

## Global Constraints

- All work happens inside `eslint-lensflow-plugin/` unless a step says otherwise (repo-root `.gitignore` and `.github/` are the only things touched outside it).
- Node engines range (`package.json`): `^20.19.0 || ^22.13.0 || >=24`. CI uses Node `24.x`.
- Coverage thresholds to enforce (measured locally, see spec): statements 86%, lines 86%, branches 80%, functions 90%.
- `vitest.config.ts` currently has no `testTimeout` (defaults to 5000ms) — this must become `20000` or `vitest run --coverage` fails with mass test timeouts (verified during design).
- Existing `pool: "forks"` / `singleFork: true` in `vitest.config.ts` stays untouched — out of scope.
- Full spec: `eslint-lensflow-plugin/docs/superpowers/specs/2026-07-20-github-ci-design.md`.

Note: this install may cause npm to also adjust the existing `dependencies.eslint-visitor-keys` range (currently `^5.0.1`, while `peerDependencies.eslint-visitor-keys` requires `^4.0.0` — a pre-existing inconsistency in this file, unrelated to CI). - [ ] **Step 2: Confirm coverage currently fails with default timeout (reproduce the bug)**

```bash
npx vitest run --coverage tests/rules/require-explicit-variance.test.ts
```

Expected: at least one `Error: Test timed out in 5000ms` failure. This confirms the bug this step fixes.

- [ ] **Step 3: Raise the test timeout and configure coverage**

Edit `eslint-lensflow-plugin/vitest.config.ts` to:

```ts
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    globals: false,
    passWithNoTests: true,
    pool: "forks",
    poolOptions: {
      forks: {
        singleFork: true,
      },
    },
    testTimeout: 20000,
    coverage: {
      provider: "v8",
      reporter: ["text", "lcov", "html", "json-summary"],
      thresholds: {
        statements: 86,
        branches: 80,
        functions: 90,
        lines: 86,
      },
    },
  },
});
```

- [ ] **Step 4: Re-run the same file to confirm the fix**

```bash
npx vitest run --coverage tests/rules/require-explicit-variance.test.ts
```

Expected: `Test Files  1 passed (1)`, no timeout errors, a coverage table printed.

- [ ] **Step 5: Add the `test:coverage` script**

In `eslint-lensflow-plugin/package.json`, add to `"scripts"` (alongside the existing `test`/`test:watch`):

```json
"test:coverage": "vitest run --coverage",
```

- [ ] **Step 6: Run the full suite with coverage and confirm thresholds are met**

```bash
npm run test:coverage
```

Expected: all 4108 tests pass (`Test Files  282 passed (282)`, `Tests  4108 passed (4108)`), no threshold-failure output, exit code 0. This run takes roughly 15-20 minutes — do not interrupt it.

- [ ] **Step 7: Commit**

`package-lock.json` stays gitignored until Task 4 — only stage the two files below, not the lockfile.


### Task 5: Add the GitHub Actions CI workflow

**Files:**

- Create: `.github/workflows/ci.yml` (repo root)

**Interfaces:**

- Consumes: npm scripts `typecheck`, `build`, `lint`, `lint:oxlint` (Task 3), `format:check` (Task 2), `test:coverage` (Task 1); `eslint-lensflow-plugin/package-lock.json` (Task 4).

- [ ] **Step 1: Create the workflow directory and file**

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  ci:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: eslint-lensflow-plugin
    steps:
      - name: Checkout
        uses: actions/checkout@v7

      - name: Setup Node.js
        uses: actions/setup-node@v7
        with:
          node-version: "24.x"
          cache: npm
          cache-dependency-path: eslint-lensflow-plugin/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Typecheck
        run: npm run typecheck

      - name: Build
        run: npm run build

      - name: Lint (ESLint)
        run: npm run lint

      - name: Lint (oxlint)
        run: npm run lint:oxlint

      - name: Format check
        run: npm run format:check

      - name: Test with coverage
        run: npm run test:coverage

      - name: Upload coverage report
        if: always()
        uses: actions/upload-artifact@v7
        with:
          name: coverage-report
          path: eslint-lensflow-plugin/coverage/
          retention-days: 14
```

- [ ] **Step 2: Validate YAML syntax**
